## 为什么需要这个 Skill？

需求评审对测试同学的三大暴击：

1. **信息过载，陷阱藏太深** — 新系统需求文档界面繁杂、故事点散乱。文档刚更新完就立刻评审，根本没时间梳理，里面的坑根本来不及发现。

2. **新人开局，硬要你输出** — 小白测试刚入职，业务一片空白，突如其来的需求评审，组长还要求你提出"有建设性的问题"——压力直接拉满。

3. **完美产品经理是稀缺物种** — 一个靠谱的产品同事是团队宝藏，但宝藏不是每队都有。你得做好心理准备，学会用工具补齐差距。

---

这个 Skill 帮你充分挖掘需求痛点：无论你是要**提出问题**，还是**借痛点快速理解需求**，都能大幅缓解新工作带来的应激反应。

最重要的一点——**它还保护了你家产品同事的人身安全。** 😄
